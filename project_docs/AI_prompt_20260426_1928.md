## 2. Dynamic Contents Array (Session Injection)

**Note on Double-Injection Patch:** Previously, the Electron frontend would pass `[Style: <Genre>]` inside the `chat_history`, and the backend would *append* the `USER PROMPT` and `LOCAL SESSION STATE` as a brand new turn. This resulted in the model receiving the user's text prompt twice. This has now been **patched** in `gemini_client.py` by detecting the final user message in the history array, popping it, and natively merging the style-tagged prompt directly into the augmented prompt payload.
