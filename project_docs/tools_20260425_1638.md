# Ableton AI Toolset Documentation

**Generated on:** 2026-04-25

This document outlines all currently available tools, their schemas, dependencies, and operational status in the Ableton AI agent ecosystem.

## 1. Exposed Base Tools

These tools map closely to 1:1 remote script commands and are exposed to the AI model in `gemini_client.py`.

### `test_ableton_connection`
- **Description**: Attempts to send a JSON 'ping' message to the Ableton Server.
- **Schema**: None
- **Dependencies**: `proxy.request_state`
- **Status**: Working / Used

### `get_song_scale`
- **Description**: Retrieves whether Scale Mode is active, the root note, and the scale name (e.g., Minor, Major).
- **Schema**: None
- **Dependencies**: `proxy.request_state`
- **Status**: Working / Used

### `get_session_info`
- **Description**: Retrieves the current tracks and state in the Ableton Live session.
- **Schema**: None
- **Dependencies**: `proxy.request_state`
- **Status**: Working / Used

### `set_tempo`
- **Description**: Sets the tempo (BPM) of the Ableton session.
- **Schema**: `TempoRequest(tempo: float)`
- **Dependencies**: `proxy.request_state`
- **Status**: Working / Used (Also in `atomic_tools`)

### `get_track_info`
- **Description**: Gets detailed info about a specific track by its integer index (0-indexed).
- **Schema**: `GetTrackInfoRequest(track_index: int)`
- **Dependencies**: `proxy.request_state`
- **Status**: Working / Used

### `create_midi_track`
- **Description**: Creates a new MIDI track in Ableton Live.
- **Schema**: `TrackNameRequest(track_name: str)`
- **Dependencies**: `proxy.request_state`
- **Status**: Working / Used

### `set_track_name`
- **Description**: Sets the name of a specific track.
- **Schema**: `TrackIndexNameRequest(track_index: int, name: str)`
- **Dependencies**: `proxy.request_state`
- **Status**: Working / Used

### `create_clip`
- **Description**: Creates a new MIDI clip in a specific track and clip slot.
- **Schema**: `CreateClipRequest(track_index: int, clip_slot_index: int, length: float)`
- **Dependencies**: `proxy.request_state`
- **Status**: Working / Used

### `set_clip_name`
- **Description**: Sets the name of an existing clip.
- **Schema**: `SetClipNameRequest(track_index: int, clip_slot_index: int, name: str)`
- **Dependencies**: `proxy.request_state`
- **Status**: Working / Used

### `add_notes_to_clip`
- **Description**: Adds MIDI notes to a clip using semantic pitch names (e.g. 'C3', 'Eb2').
- **Schema**: `AddNotesRequest(track_index: int, clip_slot_index: int, notes: List[NoteSchema])`
- **Dependencies**: `proxy.request_state`, `_pitch_name_to_midi` (Helper)
- **Status**: Working / Used

### `get_browser_tree`
- **Description**: Retrieves the root level tree of the Ableton Live browser.
- **Schema**: None
- **Dependencies**: `proxy.request_state`
- **Status**: Working / Used

### `get_browser_items_at_path`
- **Description**: Retrieves a list of children items and folders at a specific path in the Ableton Live browser.
- **Schema**: `BrowserItemsRequest(path: str)`
- **Dependencies**: `proxy.request_state`
- **Status**: Working / Used

### `load_instrument_or_effect`
- **Description**: Loads a device (instrument or effect) onto a track from the browser.
- **Schema**: `LoadDeviceRequest(track_index: int, browser_path: str)`
- **Dependencies**: `proxy.request_state`
- **Status**: Working / Used

### `load_drum_kit`
- **Description**: Loads a drum kit onto a track.
- **Schema**: `LoadDrumKitRequest(track_index: int, drum_kit_path: str)`
- **Dependencies**: `proxy.request_state`
- **Status**: Working / Used

### `get_notes_from_clip`
- **Description**: Retrieves an array of all notes in a specific clip.
- **Schema**: `GetNotesFromClipRequest(track_index: int, clip_slot_index: int)`
- **Dependencies**: `proxy.request_state`
- **Status**: Working / Used

### `delete_notes_from_clip`
- **Description**: Deletes specific notes from a clip based on precisely matching both pitch and start_time.
- **Schema**: `DeleteNotesRequest(track_index: int, clip_slot_index: int, notes: List[NoteSchema])`
- **Dependencies**: `proxy.request_state`
- **Status**: Working / Used

### `get_track_devices`
- **Description**: Gets all devices loaded on a specific track.
- **Schema**: `GetTrackDevicesRequest(track_index: int)`
- **Dependencies**: `proxy.request_state`
- **Status**: Working / Used

### `get_device_parameters`
- **Description**: Gets all parameters for a device on a track to discover their names, minimums, and maximums.
- **Schema**: `DeviceIndexRequest(track_index: int, device_index: int)`
- **Dependencies**: `proxy.request_state`
- **Status**: Working / Used

### `set_device_parameter`
- **Description**: Sets the numeric value of a specific parameter by its string name on a device using normalized values (0.0 to 1.0).
- **Schema**: `SetDeviceParameterByNameRequest(track_index: int, device_index: int, parameter_name: str, value: float)`
- **Dependencies**: `proxy.request_state`
- **Status**: Working / Used

## 2. Compound Tools

These tools combine multiple simpler API calls or contain more advanced logic for quality-of-life enhancements.

### `set_track_volume_by_name`
- **Description**: Sets the volume of a track by its name in dB. Looks up the track index automatically.
- **Schema**: `SetTrackVolumeByNameRequest(track_name: str, gain_db: float)`
- **Dependencies**: `_get_track_index_by_name`, `proxy.request_state`
- **Status**: Working / Used

### `get_session_mix_status`
- **Description**: Retrieves a summary of volume/gain status for all tracks in the session in one single text block.
- **Schema**: None
- **Dependencies**: `get_session_info`
- **Status**: Working / Used

### `inject_midi_to_new_clip`
- **Description**: Finds the first empty clip slot on a track, creates a clip, and injects notes via semantic pitch names.
- **Schema**: `InjectMidiRequest(track_index: int, length: float, notes: List[NoteSchema])`
- **Dependencies**: `proxy.request_state`, `_pitch_name_to_midi`
- **Status**: Working / Used

### `sound_design`
- **Description**: Adjusts basic macro parameters for specific Ableton devices to achieve a sound design goal. Features strict validation against a "Safe Menu" of parameters to prevent runtime errors.
- **Schema**: `SoundDesignRequest(track_name: str, device_name: str, tweaks: dict)`
- **Dependencies**: `get_session_info`, `get_track_devices`, `proxy.request_state` (via `asyncio.to_thread`)
- **Status**: Working / Used

## 3. Unregistered / Backend-Only Tools

These tools were removed from the LLM prompt during the Phase 1 refactoring but are fully functional and heavily relied upon by the frontend UI and direct API endpoints.

- **`start_playback`**: Starts playback. (Working / Hidden from LLM)
- **`stop_playback`**: Stops playback. (Working / Hidden from LLM)
- **`select_track`**: Focuses a track by name. (Working / Hidden from LLM)
- **`arm_track`**: Arms or disarms a track for recording. (Working / Hidden from LLM)
- **`fire_clip`**: Plays a specific clip. (Working / Hidden from LLM)
- **`stop_clip`**: Stops a specific clip. (Working / Hidden from LLM)
- **`delete_track`**: Deletes a track by index. (Working / Hidden from LLM)
- **`delete_clip`**: Deletes a clip by track and slot index. (Working / Hidden from LLM)

*Note: These backend-only tools use native Python dictionaries instead of Pydantic schemas for data passing.*
