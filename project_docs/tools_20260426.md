# Ableton AI Toolset Documentation

**Generated on:** 2026-04-26

This document outlines all currently available tools, their schemas, dependencies, and operational status in the Ableton AI agent ecosystem. It reflects the latest Single-Shot Compiler optimizations, categorizing tools into those explicitly passed to the LLM (Exposed Base / Compound Tools) versus those reserved for internal backend operations (Unregistered).

---

## 1. Exposed Base Tools

These tools map closely to 1:1 remote script commands and are explicitly exposed to the AI model orchestrator in `gemini_client.py`.

### `get_session_info`
- **Description**: Retrieves the current tracks and state in the Ableton Live session. Used extensively for context pre-fetching.
- **Schema**: None
- **Dependencies**: `proxy.request_state`
- **Status**: Working / Used

### `create_midi_track`
- **Description**: Creates a new MIDI track in Ableton Live.
- **Schema**: `TrackNameRequest(track_name: str)`
- **Dependencies**: `proxy.request_state`
- **Status**: Working / Used

### `set_track_name`
- **Description**: Sets the name of a specific track.
- **Schema**: `TrackIndexNameRequest(track_index: int, name: str)`
- **Dependencies**: `proxy.request_state`
- **Status**: Working / Used

### `set_clip_name`
- **Description**: Sets the name of an existing clip.
- **Schema**: `SetClipNameRequest(track_index: int, clip_slot_index: int, name: str)`
- **Dependencies**: `proxy.request_state`
- **Status**: Working / Used

### `load_instrument_or_effect`
- **Description**: Loads a device (instrument or effect) onto a track from the browser.
- **Schema**: `LoadDeviceRequest(track_index: int, browser_path: str)`
- **Dependencies**: `proxy.request_state`
- **Status**: Working / Used

### `get_notes_from_clip`
- **Description**: Retrieves an array of all notes in a specific clip.
- **Schema**: `GetNotesFromClipRequest(track_index: int, clip_slot_index: int)`
- **Dependencies**: `proxy.request_state`
- **Status**: Working / Used

### `inject_midi_to_new_clip`
- **Description**: Finds the first empty clip slot on a track, creates a clip, and injects notes via semantic pitch names.
- **Schema**: `InjectMidiRequest(track_index: int, length: float, notes: List[NoteSchema])`
- **Dependencies**: `proxy.request_state`, `_pitch_name_to_midi`
- **Status**: Working / Used

### `ui_text_response`
- **Description**: A synthetic tool generated inside the schema injection, explicitly used by the LLM to return Markdown text, advice, or sound design instructions to the frontend.
- **Schema**: Synthetic dictionary `{"text": str}`
- **Dependencies**: Handled internally by `gemini_client.py` response parser.
- **Status**: Working / Used

## 2. Compound Tools

These tools combine multiple simpler API calls or contain more advanced fuzzy-matching logic for robust prompt-engineering. They are exposed to the LLM.

### `set_track_volume_by_name`
- **Description**: Sets the volume of a track by its name in dB. Looks up the track index automatically.
- **Schema**: `SetTrackVolumeByNameRequest(track_name: str, gain_db: float)`
- **Dependencies**: `_get_track_index_by_name`, `proxy.request_state`
- **Status**: Working / Used

### `mix_track`
- **Description**: A unified tool for handling volume (normalized), panning, and mute state simultaneously on a track.
- **Schema**: `MixTrackRequest(track_index: int, volume: Optional[float], panning: Optional[float], mute: Optional[bool])`
- **Dependencies**: `proxy.request_state` (`set_track_mixer`)
- **Status**: Working / Used

### `sound_design`
- **Description**: Adjusts macro parameters for specific Ableton devices to achieve a sound design goal. Features strict validation against a "Safe Menu" of parameters and uses float normalization (0.0 to 1.0). Includes fuzzy matching for track and device names.
- **Schema**: `SoundDesignRequest(track_name: str, device_name: str, tweaks: dict)`
- **Dependencies**: `get_session_info`, `get_track_devices`, `proxy.request_state` (via `asyncio.to_thread`)
- **Status**: Working / Used

## 3. Unregistered / Backend-Only Tools

These tools are fully functional and heavily relied upon by the frontend UI and direct API endpoints (`main.py`). However, to save context tokens and encourage macro-tool usage, they have been pruned from the LLM's `self.tools` list.

- **`get_session_mix_status`**: Retrieves a summary of volume/gain status for all tracks.
- **`test_ableton_connection`**: Pings the Ableton Server.
- **`get_song_scale`**: Retrieves the current scale and root note.
- **`set_tempo`**: Sets the tempo (BPM) of the Ableton session.
- **`get_track_info`**: Gets detailed info about a specific track by its integer index.
- **`create_clip`**: Creates a new empty MIDI clip.
- **`add_notes_to_clip`**: Appends MIDI notes to an existing clip.
- **`get_browser_tree`**: Retrieves the root level tree of the Ableton Live browser.
- **`get_browser_items_at_path`**: Retrieves a list of children items and folders at a specific browser path.
- **`load_drum_kit`**: Loads a drum kit onto a track.
- **`delete_notes_from_clip`**: Deletes specific notes from a clip.
- **`get_track_devices`**: Gets all devices loaded on a specific track.
- **`get_device_parameters`**: Discovers all parameters for a device on a track (used to populate the `parameter_dump.txt`).
- **`set_device_parameter`**: Sets a raw parameter value (now largely superseded by `sound_design` for LLM usage).
- **`start_playback`**: Starts playback. 
- **`stop_playback`**: Stops playback. 
- **`select_track`**: Focuses a track by name. 
- **`arm_track`**: Arms or disarms a track for recording. 
- **`fire_clip`**: Plays a specific clip. 
- **`stop_clip`**: Stops a specific clip.
- **`delete_track`**: Deletes a track by index. 
- **`delete_clip`**: Deletes a clip by track and slot index.

*Note: Many of these backend-only tools use native Python dictionaries instead of Pydantic schemas for data passing when called directly via `main.py`.*
